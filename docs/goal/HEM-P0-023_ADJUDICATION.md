# HEM-P0-023 双语医学语义冲突专家裁决包

## 裁决状态

HEM-P0-023 仍为医学语义裁决阻塞项。本文件及配套工作簿只建立可审计的专家裁决入口，不判断中文或英文哪一方正确，不修改原始病例、审核状态或医学事实。

- 范围：18条双语症状极性冲突，涉及11例病例。
- 运行时处置：`teacherReviewRequired=true` 与病例 `needs_revision` 保持不变；冲突事实不进入确定性 Patient Agent 上下文，也不参与评分。
- 问答处置：被问及时按会话语言返回自然的不确定表达；结构化日志仅记录 `reason=medical_bilingual_conflict_pending_review`、病例ID和字段ID，不记录事实值或签名。
- 禁止事项：不得自动翻译、反转、删除或批准；不得自动解除 `needs_revision`；不得以本隔离实现代替具名专家裁决。

## 可审计来源

- 冲突清单：`server/bilingualConflictQuarantine.js`，固定18个审核项ID。
- 当前双语值及复核标志：`data/patient_slots_bilingual.json`。
- 来源摘录：`data/cases.json` 中相应病例的 `sourceFacts`；若原始资料未明确覆盖该症状，工作簿明确标记“未找到直接摘录”，不得把程序补充内容冒充原始资料。
- 来源分类及诊断/评分影响：`data/hematuria_release_v14_normalized.json` 对应追踪事实。
- 病例审核状态：`data/cases.json` 的 `medicalReview.status`。

## 专家填写规则

配套工作簿的黄色列为专家输入区。每条须由具名审核人填写中文最终值、英文最终值、决定、医学依据、审核人、专科和日期，并由复核人二次复核。任何导入动作必须经过现有医学审核导入门禁；“导入状态”在本包中保持空白。

决定仅允许：保留中文、保留英文、双方修改、删除。若证据不足，应保持空白并继续隔离，不能用“暂定”绕过审核。

## 工程解除条件

只有在18条均完成具名审核与复核、医学依据可追踪、受控导入测试通过，且原始审计轨迹保留后，才能另行评估解除本阻塞。当前 PR 必须保持 Draft。
