## 2026-07-13 — 实施结果

### 设计系统、首页与病例目录

- 将既有临床色板改为 CSS 语义 Token，补齐表面、文本、边框、信息、成功、注意、危险、圆角、阴影、过渡与层级变量。
- 新增统一按钮、输入、分段选择、状态标签和卡片样式；未引入新依赖。
- 增加 `prefers-reduced-motion` 处理，并改善打印页边距与分页基础样式。
- Header 使用粘性、紧凑、可横向滚动的移动导航；根元素 `lang` 随界面语言同步。
- 首页完成中英文呈现，并收敛主/次操作层级。
- 病例目录增加病例编号/公开主诉搜索、可恢复空状态、来源信息和“未开始/进行中/已完成”文字状态；未展示诊断或隐藏答案。

### 问诊主界面

- 1280px 宽度切换为更宽的双栏工作区，1380px 以上才显示第三辅助栏。
- 移动端压缩标题、病例公开信息和次要操作；360×800 与 390×844 首屏均可见问诊输入。
- 医生/患者消息增加明确角色标签；长文本安全换行。
- 自动滚动仅在用户接近底部时执行；手动上翻后显示“有新消息·回到底部”，不强制拉回。
- 单行输入改为多行 textarea：Enter 发送、Shift+Enter 换行；空内容、初始化和生成中禁用发送，保留既有双击防重逻辑。
- 输入区使用稳定的聊天底部操作面板，AI 状态更新不会替换输入节点或夺取焦点。
- AI 连接、健康与重连提示合并为单一状态槽；日志同步改为小型标签；同一状态最多显示一个“重新连接AI”按钮。
- 语音设置弹窗增加 Escape 关闭；图标操作保留可访问名称。

### 数据面板与 360 分报告

- 查体、检验、检查与影像沿用原数据和释放规则，统一卡片、触控尺寸和“已返回”文字状态。
- 报告卡仅在原字段存在时展示 `value / unit / referenceRange`；缺失字段显示破折号，不推断为正常。
- 正常、异常、待审核、已报告状态同时使用图标和文字；`needs_review` 不会显示为正常。
- 360 分结果增加相对强项、优先改进和危险遗漏层级；进度条仍严格由原始 `score / max` 计算，并补齐 progressbar 可访问属性。
- 关键错误使用危险语义，打印时避免分项卡跨页；360 分算法、版本和核心评分输入未修改。

## 已知限制

- 当前产品基线没有深色模式，本分支未新增主题切换；浅色对比度已纳入 axe 与视觉验收。
- 仓库仍缺少完整本地训练 API 启动器，因此手工浏览器无法真实推进第 2–7 阶段；这些阶段通过拦截式 Playwright、42 例回归和源码结构验收。
- 本机依赖运行时为 Node 24.14，项目声明 Node 22；命令会给出 engine warning，但所有门禁与构建均成功。CI 应继续使用项目声明的 Node 22。
