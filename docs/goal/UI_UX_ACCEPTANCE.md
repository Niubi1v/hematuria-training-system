# UI/UX 验收标准

## 通用视觉系统

- [ ] 页面、表面、边框、文本、主色和语义色均由统一 Token 驱动。
- [ ] 成功只使用绿色，注意/降级使用黄色，危险/失败使用红色，信息使用蓝色。
- [ ] 同类按钮、卡片、标签、提示和输入控件视觉一致。
- [ ] 不引入新 UI 框架或高成本持续动画；支持 `prefers-reduced-motion`。
- [ ] 中文与英文均不截断、不造成水平滚动；根元素 `lang` 与界面语言同步。

## 问诊主界面（最高优先级）

- [ ] 1440×900、1280×720、390×844、360×800 均无横向滚动、遮挡或屏外按钮。
- [ ] 标题、公开病例信息、聊天、输入和阶段入口层级明确；隐藏病史和诊断不泄露。
- [ ] 1280×720 与移动端首屏可直接看到患者对话和输入区。
- [ ] 医生/患者消息有文字与视觉双重区分；长中英文可换行。
- [ ] 只有用户接近底部时自动滚动；手动上翻后不强拉回。
- [ ] 多行输入支持 Enter 发送、Shift+Enter 换行；发送中禁用，快速双击不重复发送。
- [ ] AI 状态变化不导致输入失焦或输入区位移。
- [ ] ready 不出现大横幅；connecting/reconnecting 只有一个轻提示；degraded 只有一个提示和一个“重新连接AI”。
- [ ] 规则库来源仍明确为 fallback，不伪装为 live AI；日志同步使用次级小标签。

## 病例目录与导航

- [ ] 可按病例编号/公开主诉搜索，并保留难度与来源筛选。
- [ ] 无搜索结果有明确、可恢复的空状态。
- [ ] 卡片不展示诊断、隐藏病史或医学答案。
- [ ] 已完成、进行中、未开始状态包含文字，不仅依靠颜色。
- [ ] 当前步骤、主操作和次操作在桌面与移动端都清楚；刷新后沿用既有状态恢复机制。

## 查体、检验、检查与影像

- [ ] 未检查、等待、正常、异常、待审核均有文字/图标，不仅靠颜色。
- [ ] 不将缺失值显示为正常；`needs_review` 保持明确。
- [ ] 数值、单位、参考范围与长文本安全换行；移动端使用卡片或安全滚动容器。
- [ ] 数据内容、获取规则、审核状态和医学契约不变。

## 评分结果

- [ ] 总分、分项、强项、改进项、危险遗漏和关键错误层级清楚。
- [ ] 进度条宽度严格由 `score / max` 计算；360 分算法与版本不变。
- [ ] 医学待审核项不呈现为确定结论。
- [ ] 中文/英文布局一致；打印时隐藏交互控件且内容不被截断。

## 可访问性与质量门禁

- [ ] 可见焦点、合理 Tab 顺序、表单 label、按钮名称和图标替代文本完整。
- [ ] 弹窗可用 Escape 关闭，焦点不丢失。
- [ ] 关键触控区域最小 44×44px（紧凑内联文本操作除外）。
- [ ] Playwright 结构/交互测试、axe 关键页扫描、console error 检查通过。
- [ ] TypeScript、ESLint、构建、42 例回归、医学门禁、安全扫描通过。
- [ ] 修改前后截图使用相同视口、语言和状态保存于 `docs/goal/screenshots/`。
