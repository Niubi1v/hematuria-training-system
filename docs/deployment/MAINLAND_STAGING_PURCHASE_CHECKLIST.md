# 中国大陆预发布购买与开通清单

本文仅供用户在腾讯云控制台人工执行。脚本不会购买资源、创建账号、修改 DNS、申请备案或写入真实密钥。

## 购买前门禁

- [ ] 已确认专项分支的构建、TypeScript、ESLint、Playwright、secret/bundle 扫描和 `data/**` 零差异。
- [ ] 已保存本次测试报告、镜像标签和 Git SHA。
- [ ] 已确定主要教师所在区域，并优先选择网络距离较近的腾讯云地域。
- [ ] 已确认域名主体、实名认证和备案责任人。
- [ ] 已确定预算、数据保留期、告警接收人及回滚责任人。

## 控制台人工步骤

1. [ ] 选择 CVM 地域；先比较北京/华北、上海/华东、广州/华南到主要教师的网络表现。
2. [ ] 创建同地域 VPC 与私有子网。
3. [ ] 创建 CVM，启用云硬盘快照与基础监控。
4. [ ] 在同账号、同地域、同 VPC 创建腾讯云 Redis。
5. [ ] Redis 白名单只允许应用子网或 CVM 私网地址；不分配公网入口。
6. [ ] CVM 安全组只开放必要入口：SSH（仅固定运维源地址）、80、443。
7. [ ] 不开放 Node 3000、Redis 6379、mock 8787 到公网。
8. [ ] 创建独立预发布域名，不与 Vercel Preview 或未来 Production 共用 token、Redis namespace 或密钥。
9. [ ] 完成 ICP 备案/接入备案要求后再提供大陆公网服务。
10. [ ] 人工添加 DNS 解析；本仓库不修改 DNS。
11. [ ] 申请并安装 HTTPS 证书，验证完整链、自动续期和到期告警。
12. [ ] 配置 CVM、Redis、带宽、磁盘、5xx、健康 503、进程重启告警。
13. [ ] 配置 Redis 自动备份，并执行一次人工恢复演练。
14. [ ] 在 CVM 上创建权限为最小化的 `.env.mainland`，写入真实服务器变量；不得上传到 Git。
15. [ ] 部署不可变镜像，执行深度 health、20 轮 safe_mock，再经授权执行真实 DeepSeek 验收。
16. [ ] 保存部署 manifest，验证回滚到上一镜像及 Redis 恢复点。

## 资源规格与扩容门槛

当前代码库是单实例 Next.js + Nginx，Redis 为托管服务。本地 Compose 空闲/功能测试观测值为：应用约 75 MiB、Nginx 约 31 MiB、Redis 约 10 MiB、safe_mock 约 20 MiB，总计约 136 MiB；应用镜像约 191 MB。中英文各 20 轮功能会话与 Redis 并发幂等测试通过，但这不是生产并发峰值容量证明，购买后仍须采集真实 P95。

- 最低试运行候选：2 vCPU / 4 GiB CVM、系统盘 50 GiB；仅用于少量受控教师试用，不作为既定结论。
- 推荐预发布候选：4 vCPU / 8 GiB CVM、系统盘 80 GiB；为构建、滚动重启和观测预留余量。
- Redis：从腾讯云可用的最小受支持高可用规格起步，容量至少为压测峰值 key/value 占用的 3 倍，且启用备份。不要仅凭本文选择大型规格。

满足任一条件时评估纵向扩容或增加应用实例（增加实例前必须先把 agent/TTS admission store 从单机内存迁移为共享 Redis）：

- 15 分钟窗口 CPU P95 持续超过 70%；
- 运行时内存 P95 超过可用内存 70%，或发生 OOM/频繁重启；
- API/AI 完整回答 P95 超过验收阈值且 provider 延迟不是主因；
- 5xx 超过 1% 或健康 503 持续 2 分钟；
- Redis 内存超过 60%、连接数超过配额 70%、命令 P95 持续恶化；
- 20 轮成功率低于 99%，或 fallback 率超过已批准阈值。
